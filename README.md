О проекте

Основная идея проекта заключается в создании автоматизированного конвейера данных (Data Pipeline) для сбора, централизованного хранения, трансформации и визуализации статистики из шахматных онлайн-платформ (Lichess, Chess.com).

Источниками данных являются два самых популярных сервиса для игры в шахматы - Lichess, Chess.com. Оба сервиса поддерживают выгрузку информации по API.
За оркестрацию отвечает Airflow - управляет расписанием вызовов API, загрузкой и трансформацией данных внутри хранилища.
Данные из API в неизменном ввиде попадают в data lake, реализованное с помощью Minio. После данные переводятся в ODS хранилище для оптимального  хранения. После чего по необходимости данные преобразуются под визуализацию в витрины данных. Из витрин данные поступают в Apache Superset для построения дашбордов.

Как запустить

- Клонировать проект
- Создать venv виртуальное окружение
- Для редактирования нужно в виртуальное окружение загрузить:
    Airflow=2.10.5 (или более позднюю версию 2.11.*)
    apache-airflow-providers-amazon==9.2.0   
- Поднять контейнеры с docker-compose

    Полезнае команды
    python -m venv venv - создать виртуальное окружение
    venv\\Scripts\\activate - запуск виртуального окружения
    docker-compose down -v - остановить контейнеры докер
    docker-compose up -d  - запустить контейнеры докер

- Зайти в MinioUI (логин/пароль в docker-compose файле)
- Создать bucket, access/secret ключи
- Зайти в AirflowUI (логин/пароль в docker-compose файле)
- В airflow ui прописать переменную lichess_user, chess_user
- Во вкладке Connections создать соединение с minio, указать access/secret ключи

Описание методов, классов, дагов

Dags:
    load_raw_data_from_lichess_to_s3.py - Даг для запуска загрузки файла из API Lichess.com за вчерашний день
        Функции:
            - load_raw_data_from_lichess_to_s3 - основная функция, вызывается в task2 с помощью PythonOperator. Вызывает выгрузку файла из API и инициализирует загрузку файла в s3 хранилище
            - get_file_from_lichess - функция для загрузки файла из Lichess за вчерашний день. Вызывается внутри функции load_raw_data_from_lichess_to_s3. На вход получает время начала и окончания загрузки, определяющие диапозон за которые будут получены данные. Возвращает строку с текстом полученным из файла.
            - get_loading_time - функция для получения диапазона загрузки файла. Вызывается внутри функции load_raw_data_from_lichess_to_s3. Использует переменную data_interval_end из контекста Airflow для определения даты загрузки. Возвращает время начала и окончания загрузки
            - load_data_to_s3 - функция для отправки файла в s3 Minio хранилище. Вызывается внутри функции load_raw_data_from_lichess_to_s3

    load_raw_data_from_chess_com_to_s3.py - Даг для запуска загрузки файла из API Chess.com за текущий месяц
        Функции:
            - load_raw_data_from_chess_com_to_s3 - основная функция, вызывается в task2 с помощью PythonOperator. Вызывает выгрузку файла из API и инициализирует загрузку файла в s3 хранилище
            - get_file_from_chess_com - функция для загрузки файла из Chess.com за текущий месяц. Вызывается внутри функции load_raw_data_from_chess_com_to_s3. На вход получает месяц и год загрузки, определяющие диапозон за которые будут получены данные. Возвращает строку с текстом полученным из файла.
            - get_loading_time - функция для месяца и года загрузки, преобразование месяца в формат mm (важно для работы API). Вызывается внутри функции load_raw_data_from_chess_com_to_s3. Использует переменную data_interval_end из контекста Airflow для определения даты загрузки. Возвращает месяц в формате mm и год (int)
            - load_data_to_s3 - функция для отправки файла в s3 Minio хранилище. Вызывается внутри функции load_raw_data_from_chess_com_to_s3

Classes:
    MinioLoader - Класс для стандартизации загрузки в Minio хранилище
        Атрибуты:
            hook - соединение с бд
            bucket - название bucket
        
        Методы:
            upload - метод, запускающий отправку файла с Minio хранилище. На вход получает файл, адрес, куда нужно сохранить файл, флаг replace, определяющий, нужно ли заменить уже имеющийся файл (по умолчанию True)
